Projeto Agenda - Primeiro Deploy

Este projeto consiste na elaboração e deploy de uma aplicação simples, cujo objetivo é demonstrar a integração do Node utilizando-se o micro-framework Express com a base de dados MongoDB e o servidor remoto de cloud computing (GCP) e o proxy-reverso com Nginx.


Abstração
    No servidor faremos as conexões necessárias, as importações dos pacotes as configurações e a escolha da porta, então iremos para o arquivo de rotas onde criaremos as conexões com os controladores e se necessário com os middlewares, lembrando que cada controlador fica responsável por uma classe de rota, suas ações dependerão do tipo de requisição e para cada tipo poderemos ordenar e executar funções, funções estas cuja origem se da no arquivo Model. O arquivo model é um dos mais importantes neste projeto pois nele está toda dinâmica necessária para a aplicação fluir é lá onde desenvolvemos o algoritmo que fará a interação com usuário e o banco de dados, no entanto não podemos esquecer do Mongoose, um excelente recurso que através de seus métodos permite a busca e inserção de dados entre o Node e o MongoDB, e para visualizarmos toda esta ópera temos o View que é acionado pelo controller dependendo do retorno de cada requisição, no view temos as páginas HTML renderizadas com o recurso de mesmo nome "view" utilizando-se do motor EJS, esta engine nos autoriza a inserir JavaScript diretamente no arquivo HTML. No views teremos as interações das variáveis locais geradas pelo Models e middlewares com os objetos da requisição. Por fim, os arquivos Front-end no qual pecamos em não utilizar todo seu potêncial pois é nele onde podemos economizar muitos recursos e minimizar ao máximo a interação entre o back-end e o banco de dados. 


Cloud Computing e Deploy
    Bom após desenvolver toda dinâmica necessária entre a aplicação e o banco de dados, e testar seu funcionamento no host local iremos agora subir estes dados, utilizando inicialmente o Google Cloud Platform que após configurado nos dará um servidor remoto. Utilizando SSH iremos nos conectar com o nosso novo servidor, então com git criaremos e configuraremos duas pastas, uma de repositório que receberá as atualizações feitas localmente e outra de distribuição, feita o versionamento da aplicação local com a aplicação remota, o passo final é fazer o Proxy Reverso utilizando Nginx, este recurso vai fazer o deploy da nossa aplicação colocando-a online, para que isso funcione precisamos apenas instalar o Nginx e configurá-lo um processo simples, de 5 minutos.
